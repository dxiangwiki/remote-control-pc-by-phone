# 局域网手机遥控电脑网页项目
✅ 无需同一网页 ✅ 只需共享数据 ✅ 纯原生实现 ✅ 工程化规范

## 核心架构观点（作者设计思想）
多端联动、设备交互，**不需要使用同一个网页、同一个服务**，
真正的核心只有一个：**共享同一组数据（轻量级数据中心）**。

页面只是载体，数据才是灵魂。
—— 来自一位注重架构与工程化的学生开发者

## 项目介绍
本项目实现：**局域网内，手机浏览器隔空控制电脑网页**

- 手机端：独立遥控器页面
- 电脑端：独立展示受控页面
- 服务端：统一数据中转中心
- 两端完全解耦，互不耦合污染

## 技术栈
- Node.js + Express 后端服务
- 原生 HTML / JavaScript 无框架开发
- 基于 HTTP 接口跨端通信
- 高性能优化：仅数据发生变化时才渲染，无变化不操作 DOM

## 项目结构
```
remote-control-pc-by-phone/
├─ server/          # 后端中转服务
├─ client-phone/    # 手机遥控端页面
└─ client-pc/       # 电脑展示受控页面
```

## 快速运行教程
### 1. 初始化服务端依赖
```bash
cd server
npm init -y
npm install express cors
```

### 2. 启动后端服务
```bash
node index.js
```

### 3. 局域网访问页面
电脑端：`http://你的局域网IP:3000/client-pc/index.html`
手机端：`http://你的局域网IP:3000/client-phone/index.html`

## 核心设计亮点
1. **前后端彻底解耦**：两端页面独立部署，共用一套数据中心
2. **高性能渲染机制**：数据比对节流，减少不必要 DOM 操作
3. **标准化工程化**：小型项目也遵循 npm 依赖管理规范
4. **极易扩展**：可自由新增遥控按钮、控制逻辑、页面样式

## 作者核心开发理念
1. 写代码先搭工程化架构，再写业务逻辑
2. 坚持数据驱动视图，而非页面驱动交互
3. 数据比对开销远小于 DOM 渲染，能比对就不重复渲染
4. 页面可以拆分、业务可以拆分，核心数据必须统一
